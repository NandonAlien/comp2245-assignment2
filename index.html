October 25,2022 by Norris Isaac

Usually I would have posted this a day earlier, but I had some troubles with preventing work piling up and had to postpone it from yesturday to today. University work might be difficult but with enough time and thinking it can be completed. Not much has changed besides the weather getting more colder as it reaches the end of 2022 which is fine since that heat wave was a pain. Today I will talk about the usage of physics in engines for my newest game. The Game I've been developing, Traverser has been going well following the process I came up with last month and I found I can go and do the optional sides of it. Firstly I need make the game switch to a special scene that would be a platforming mini game and since I'm new to platformers I would look at some tutorials. I've notice that there has been a focus on simulating physics concepts such as gravity. This is there where I stumbled down the rabit hole of physics engines.

Luckily, I'm using an engine that handles the image vector work but it really make me realise where difficulties in making engines usually arise and the mathematical requirements for making 3D games. I've mainly used a few variables that store the properties needed. Gravity is that first variable with it being similar to acceleration I use for my top down games however it is constantly activated on the real time loop function that keeps updating the character to go down. There is also several thing done like limiting the controls to horizontal instead of both with a jump function that only causes you to jump when you are on the ground. I would go over more of it in my Semi Annual Devlog this November. Things are definitely different since this blog started last November and how it will end up in the future might be something great.
